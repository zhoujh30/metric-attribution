# AB 实验归因分析技能

## 概述

本技能分为两个阶段：**需求对齐（Phase 0）** 和 **分析执行（Phase 1-5）**。Phase 0 必须在写任何代码之前完成——先和用户对齐"分析什么、怎么分析、分析完干什么"，再进入开发。

---

## Phase 0: 需求对齐（必须先完成）

**原则：不要拿到数据就开始跑代码。先花 1-2 轮对话把分析方向锁死。**

### 0.1 数据快速探查

拿到数据后，先静默地做一次快速探查（不需要展示给用户）：
- 读取表结构：列名、数据类型、行数
- 识别实验组/对照组标识列及其取值
- 识别核心指标列（PV/Click/Order/GMV/Subsidy）
- 识别可下钻维度列（类目层级、价格段、用户类型、商家属性等）
- 计算全局大盘指标（实验 vs 对照的 PV/GMV/GPM 总量）

目的是用数据事实支撑后续的需求对话，而非盲目提问。

### 0.2 与用户对齐三件事

基于探查结果，用 **一轮对话** 和用户对齐以下三件事。用 `ask_user_input` 工具将选项结构化呈现，降低用户回答成本。

#### ① 研究问题（我们要回答什么？）

先用数据说话，给出初步发现，例如：
> "数据显示实验组 PV +29.5% 但 GMV -1.4%，GPM 从 35.5 跌至 27.0。"

然后让用户确认或修正核心问题，常见问题类型：
- **效率归因型**："为什么流量涨了但 GMV 没涨？是分发问题还是转化问题？"
- **负向定位型**："哪些商品 / 类目 / 维度组合在拖后腿？"
- **策略评估型**："当前策略对不同用户群体的影响分别是什么？"
- **综合诊断型**：以上全要

如果用户需求不明确，主动提出建议方向；如果用户有明确假设（如"我怀疑是高价品转化出了问题"），将其作为优先验证的假设纳入分析。

#### ② 分析角度（从哪些维度切入？）

基于数据中实际存在的维度列，让用户确认需要下钻的维度。展示数据中可用的维度及其取值范围，例如：
> "数据中有以下可下钻维度：一级类目（15 个）、二级类目（45 个）、三级类目（120 个）、价格段（7 档）、用户类型（3 类）、商家层级（T1-T5）、商家类型（Local/CB）。"

默认推荐的标准分析角度：
- 类目层级（一级→二级→三级逐层下钻）
- 价格段
- 用户类型
- 商家层级
- 维度组合（类目×价格段）

用户可能会：
- 增加维度（如"还想看平台补贴的效率"）
- 删减维度（如"商家层级不重要，不用看"）
- 指定交叉（如"我特别想看用户类型×价格段的交叉"）

#### ③ 行动策略方向（分析完要干什么？）

分析报告的落脚点取决于后续行动，不同行动方向会影响输出重点。常见方向：
- **选品调整**：需要 A 类特征提取（扩品白名单）+ D 类黑名单（附 product_id 清单）
- **算法/策略调优**：需要 GPM Mix-Rate 拆解 + pCVR 校准建议 + 分用户群策略建议
- **流量分配优化**：需要按象限量化流量效率 + 流量回流模拟
- **补贴效率审计**：需要单位 PV 补贴成本 + ROAS 对比 + 补贴浪费定位
- **综合报告（汇报用）**：以上全要，侧重可视化和结论清晰度

让用户选择 1-2 个核心方向，据此调整后续输出的侧重点。

### 0.3 确认分析计划

将以上三件事总结为一段简洁的分析计划，让用户确认后再开始执行。例如：

> **分析计划确认：**
> - **核心问题**：实验组 PV +29.5% 但 GMV -1.4%，需定位 GPM 衰减根因（分发 vs 转化）
> - **分析维度**：类目三级穿透 + 价格段 + 用户类型 + 商家层级 + 维度组合（类目×价格段）；商家层级不看补贴效率
> - **行动方向**：选品调整（A 类扩品 + D 类剔除名单）+ 算法策略调优建议
> - **输出形式**：HTML 交互看板 + D 类黑名单 Excel

用户确认后，进入 Phase 1。

## Phase 1: 数据处理与指标计算

### 1.1 数据粒度确认

同一 product_id 可能按 user_type、price_range 等维度拆分多行。需根据分析需要决定聚合方式：
- 商品级分析（四象限分类）：按 product_id 聚合全部指标
- 维度交叉分析（如用户类型表现）：保留原始粒度

```python
# 商品级聚合
agg_exp = df[df['version_id']==exp_id].groupby('product_id')[metric_cols].sum()
agg_ctrl = df[df['version_id']==ctrl_id].groupby('product_id')[metric_cols].sum()
merged = agg_exp.join(agg_ctrl, lsuffix='_exp', rsuffix='_ctrl', how='inner')

# 获取商品属性（类目、价格段等，取去重后的第一行）
attrs = df[df['version_id']==exp_id].drop_duplicates('product_id').set_index('product_id')[attr_cols]
merged = merged.join(attrs)
```

### 1.2 指标体系

| 层级 | 指标 | 公式 |
|------|------|------|
| 规模 | PV, GMV, Orders, Subsidy | 直接求和 |
| 转化漏斗 | CTR, CO | Click/Show, Order/Click |
| 变现效率 | GPM, OPM, AOV | GMV/PV×1000, Order/PV×1000, GMV/Order |

**展示原则：相对变化率（%）大字优先，绝对值小字辅助。** 例如 "PV +29.5%" 为主显示，"(+321,752)" 为辅助信息。

## Phase 2: 四象限分类

按商品级别的 PV 变化和 GMV 变化方向分四类：

| 象限 | PV | GMV | 含义 | 分析重点 |
|------|-----|------|------|---------|
| A 类 | ↑ | ↑ | 双增 (标杆) | 提取正向特征 → 扩品池 |
| B 类 | ↓ | ↑ | 提效 (存量优化) | 看转化提升点 |
| C 类 | ↓ | ↓ | 双跌 (存量萎缩) | 是否被流量"劫持"？补贴是否退坡？ |
| D 类 | ↑ | ↓ | 高曝低转 (流量浪费) | **核心审计点** — 为什么流量涨但不转化？ |

```python
def classify(row):
    pv_up, gmv_up = row['pv_diff'] > 0, row['gmv_diff'] > 0
    if pv_up and gmv_up: return 'A'
    elif not pv_up and gmv_up: return 'B'
    elif not pv_up and not gmv_up: return 'C'
    else: return 'D'
```

**每个象限必须输出：** PV（涨幅%）、GMV（涨幅%）、CTR、CO、GPM、AOV、补贴。

重点对比 C/D 类的 GPM 和 AOV 变化，确认"客单价降了"还是"根本没人买"：
- CO 大跌 + AOV 温和 → 转化出了问题（没人买）
- CO 温和 + AOV 大跌 → 客单价断层（买的便宜了）

## Phase 3: 负向来源多维下钻

对 C/D 类（以用户在 Phase 0 确认的维度为准）进行穿透，**每个维度按负向 GMV 贡献排序**。

### 标准维度菜单

| 维度 | 分析目的 | 关键问题 |
|------|---------|---------|
| 类目（一级→二级→三级）| 定位出血品类 | 哪些三级类目在流血？ |
| 价格段 | 价格匹配度 | 是否高价断层或低价引流？ |
| 用户类型 | 流量匹配度 | 新用户被导到了不感兴趣的品？ |
| 商家层级 | 供给质量 | 流量是否涌向低层级商家？ |
| 维度组合（类目×价格段）| 特征交叉定位 | 负向 GMV 的 Top10 组合是什么？ |

**下钻表格标准列：** 维度值、SKU 数、ΔGMV、PV 涨幅%、GPM 实验/对照、GPM 跌幅%、CO 实验/对照

可根据 Phase 0 中用户追加的维度灵活扩展（如补贴效率、seller 维度等）。

---

## Phase 4: GPM Mix-Rate 效应拆解

**定性结论的关键——区分"结构性问题"和"效率性问题"。**

```
GPM 变化 = Mix Effect（流量结构变化） + Rate Effect（同类目转化率变化）

Mix Effect  = Σ (share_exp_i - share_ctrl_i) × gpm_ctrl_i
Rate Effect = Σ share_exp_i × (gpm_exp_i - gpm_ctrl_i)
```

| 结果 | 含义 | 典型行动 |
|------|------|---------|
| Rate Effect 为主 | 同类目下转化率全跌，算法效率问题 | 审计 pCVR 模块、调整分发逻辑 |
| Mix Effect 为主 | 流量结构偏移到低 GPM 类目 | 调整类目间流量分配权重 |
| 两者兼有 | 复合问题 | 分层治理 |

按一级类目拆解输出每个类目的 Mix 和 Rate 贡献。

## Phase 5: 行动方案输出

**行动方案的内容和侧重取决于 Phase 0 中确认的行动方向。** 以下是各方向的标准输出模块，按需组合：

### 模块 A: 选品调整

**A 类特征提取（扩品池）：**
- 共性画像：核心类目分布、价格段分布、商家层级、商家类型
- 正向贡献排序：按一级/三级类目、维度组合排列
- 扩品建议：优先补充哪些三级类目、锚定什么价格段、选什么商家

**D 类黑名单（Excel 清单）：**
- 全量 D 类商品按 ΔGMV 排序
- 严重程度分三档：
  - 立即剔除：对照有成交但实验零成交
  - 强烈建议剔除：双组均零成交（纯浪费）
  - 限流观察：有成交但 GMV 负增长
- 共性特征总结 → 转化为未来选品规避规则

### 模块 B: 算法/策略调优

- pCVR 校准建议（哪些类目预估失准）
- 分用户群策略建议（老用户 vs 新用户差异化分发）
- 转化门槛建议（如高价非刚需品需有偏好信号才分发）

### 模块 C: 流量分配优化

- 各象限的 PV 效率量化（每千 PV 产出多少 GMV）
- 流量回流建议（D 类释放的 PV 应回流给哪些 A/B 类品）

### 模块 D: 补贴效率审计

- 单位 PV 补贴成本对比
- 按象限和商家层级的 ROAS 对比
- 高风险浪费标记

### 模块 E: C 类治理

- 诊断：补贴退坡 vs 流量被劫持
- 恢复策略：保底流量池、补贴恢复、保护品标记

### 优先级框架

所有建议按 P0-P3 排序：
- **P0 紧急止血**：剔除零成交品 + 限流 Top 负向品，释放 PV
- **P1 根因修复**：审计模型 / 调整分发逻辑
- **P2 扩品增长**：基于 A 类特征扩充品池
- **P3 长期建设**：监控看板 + 分发策略重建

## 输出规范

### HTML 交互看板

使用纯 HTML + CSS + Chart.js（CDN），不使用 React。

**结构：**
- Header：实验名称、实验组/对照组 ID、核心 KPI 一行摘要
- Tab 导航（sticky）：全局概览 | 四象限分析 | 负向深度下钻 | GPM 拆解 | 行动方案
- 下钻 Tab 内使用 sub-tab 切换维度
- 行动方案 Tab 内使用 sub-tab 切换模块

**设计规范：**
- 深色主题（bg: #0c0e14, card: #161923）
- 字体：JetBrains Mono（数据） + Noto Sans SC（中文）
- 颜色语义：A=#10b981, B=#3b82f6, C=#eab308, D=#ef4444
- 涨跌色：up=#34d399, down=#f87171
- 指标卡片：百分比大字 + 绝对值小字
- Insight 框：danger/warn/success/info 四种
- 表格前 3-5 行高亮（最严重的行）

### Excel 清单（单独文件）

D 类全量黑名单，包含：product_id、ΔGMV、PV 变化%、GPM 实验/对照、一/二/三级类目、价格段、商家层级/类型、处置建议（分三档）。

根据用户需求，也可输出 A 类白名单 Excel。

## 话术模板

**GPM 归因结论：**
- Rate 主导：「这是算法转化效率的系统性问题，不是流量分发错位。Rate Effect 贡献了 X% 的 GPM 衰减。」
- Mix 主导：「流量结构显著偏移，更多流量流向低 GPM 类目。Mix Effect 贡献了 X% 的 GPM 衰减。」

**C/D 类判断：**
- 转化问题：「核心是"根本没人买"而非"客单价变低"。D 类 CO -X%，AOV 变化温和。」
- 客单价问题：「问题在于客单价断层。D 类 AOV -X%，低价品占据了大量流量。」
